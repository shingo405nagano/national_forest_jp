# national_forest_jp

このプロジェクトは、G空間センターにて公開されている、農林水産省林野庁の国有林小班区画データをダウンロードし、森林計画区ごとに扱いやすい GeoDataFrame に変換するための Python ライブラリです。ライブラリでは、次のような機能を提供しています。

- 都道府県単位でデータを取得する
- 森林計画区ごとのファイルを見つける
- GeoDataFrame を取得して整形する
- コード値をエンコード / デコードする
- 属性名と別名の対応を確認する

## 主な機能

- `nfj.geospatial.GsicAddressShape` で国有林の小班区画データを扱うインスタンスを作成する
- `geodataframe()` メソッドで森林計画区の GeoDataFrame を取得する
- `encode()` / `decode()` メソッドでコード値とラベルを相互変換する
- `summary()` メソッドでデータ全体の概要を YAML / JSON / Python の辞書として取得する
- `field_and_alias()` メソッドで属性名と日本語別名の対応を確認する

## セットアップ

依存関係は `requirements.txt` に、パッケージ情報は `pyproject.toml` に定義しています。

```bash
pip install -r requirements.txt
pip install -e .[dev]
```

Python は 3.13 系で動作確認しています。

## 実行前提

- `GsicAddressShape` は設定ファイル `nfj/.confs/urls.yaml` に定義された URL からデータを取得します。
- そのため、`geodataframe()` や `summary()` を使うときはネットワーク接続が必要です。
- 開発や検証では `pytest` を実行するだけでテストを回せます。

## テスト実行

テストは `pytest` を実行するだけで動きます。

```bash
pytest
```

カバレッジレポートは `nfj/tests/coverage_report/report.html` に出力されます。

## 使い方

### 基本例

```python
from nfj.geospatial import GsicAddressShape

# 都道府県を指定して`G空間センター`から国有林の小班区画データをダウンロード
# データはZip形式で公開されているものです。
shp = GsicAddressShape(prefecture="滋賀県")

# ダウンロードしたデータから森林計画区域の名前を取得
print("森林計画区域の名前:")
print(shp.plan_area_names)

# 指定した森林計画区域に対応する小班区画のGeoDataFrameを取得
# この時点で様々な前処理が行われ、属性名の正規化などが行われています。
gdf = shp.geodataframe(plan_area="湖南森林計画区")
print(gdf.shape)

# コード値をエンコード / デコードする例
encoded_gdf = shp.encode(gdf)
decoded_gdf = shp.decode(encoded_gdf)

# Field(en)とAlias(ja)の対応を確認する例
print("Field と Alias の対応:")
for field, alias in shp.field_and_alias().items():
    print(f"  {field} -> {alias}")

shp.cleanup()
```

## データについて

本プロジェクトは `nfj/.confs/urls.yaml` に定義された URL からデータを取得します。

上記のURLは [G空間センター](https://front.geospatial.jp/) が公開している、農林水産省林野庁の国有林小班区画データです。

現状「国有林GISデータ（2024（令和６）年度版）」を2025年12月に設定していますが、`urls.yaml` に定義された URL を変更すれば、最新のデータを取得することも可能です。

出典：国有林 GIS データ（https://www.geospatial.jp/ckan/dataset/nfgis-2024）、PDL1.0（https://www.digital.go.jp/resources/open_data/public_data_license_v1.0）

 - [データ仕様書](https://www.geospatial.jp/ckan/dataset/nfgis-2024/resource/f4c248b6-791b-4a7c-b968-65191b1a6491)
 - [コード表](https://www.geospatial.jp/ckan/dataset/nfgis-2024/resource/f4c248b6-791b-4a7c-b968-65191b1a6491)
 - [国有林GISデータの利用に当たっての留意事項](https://www.geospatial.jp/ckan/dataset/nfgis-2024/resource/7c9cccb2-ffab-4964-82ee-c388ad026c3c)

## データの加工について

このライブラリで取得する GeoDataFrame は、次のような前処理が行われています。

 1. EsriShapefile を GeoDataFrame として読み込む。
 2. GeoDataFrame のカラムを英名に変換し、必要なカラムのみを残す。
 3. 各カラムのデータ型を '.confs/fields.yaml' に定義されたものに基づいて適切に変換する。
 4. Nan 値を適切なデフォルト値に置換する。
 5. 国有林名に全角数字が含まれるため、全角を半角に変換する。
 6. 林小班名に余計な文字が含まれる為、削除してしまう。
 7. 樹立年度を和暦から西暦に変換する。
 8. 樹齢を修正する。
 9. 保護林の区分コードを正式な名称に変換する。
 10. 緑の回廊の区分コードを正式な名称に変換する。
 11. 重複する林小班が存在する場合は、1つにまとめる。
 12. Geometry のバリデーションを行い、無効なジオメトリを修正する。

これらは公開されているデータをダウンロードした後に、ライブラリ内で自動的に行われる処理です。ユーザーはこれらの前処理を意識することなく、整形された GeoDataFrame を利用できます。
